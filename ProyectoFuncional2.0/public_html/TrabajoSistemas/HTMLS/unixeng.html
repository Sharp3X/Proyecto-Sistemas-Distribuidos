UNIX

The memory management is highly dependent on hardware support available, there are different forms of memory management in UNIX. Current implementations provide virtual memory demand paging combined with swapping process.

For each entry in the page table is considered the physical address of the page frame, protection bits (r, w, x), validity bit, reference bit, modified page bit, and a counter-old page . Only the physical address and the validity bit must be necessarily supported by hardware, the rest of the information can be simulated in software.

In addition, for each entry in the page table stored information to access the page in secondary memory (disk block descriptor). Other information structure that remains is a list of page frames, which stores information about the status of the page (assigned, executable file, free, charging DMA), the number of processes that reference frame (for sharing) and the device containing a copy of the page and the block number.

Some characteristics of the virtual memory management are:

  • Global Allocation and working-set. UNIX defines the working-set of a process as a set of pages allocated to a process. Frames are not assigned to any process free frames are defined by the system, assignable to any process. If in a moment of time the limit is reached, a pager comes to age pages. Any reference to a page reset your age. If the age reaches a value n, established by the installation, the page goes to idle. This mechanism (clock algorithm) is also used by Linux.
  • Writing pages in disk by groups. If the pager decided to release a page and the page has changed or there is no copy in the disk, this puts the page in a list of pages to write. If the number of items in this list reaches a certain limit, the pages are physically written to disk.
  • Swapping: In situations high demand memory by processes when the page fault routine is not able to maintain the minimum level of free frames, the swapper process enters a process to remove memory. The selection criteria depend on the version of UNIX, but mainly rely on the state of the process and its priority.
