Linux

Linux uses a page table with three levels, each table is the size of a page:
  • Website Directory: Active processes have a unique page directory in the page size. The page directory must reside in main memory for all active process.
  • Intermediate Website Directory: This directory spans itself into multiple pages. Each entry in the page table points to a page that contains a page table.
  • Table of Pages: Each entry in the table refers to a virtual page of the process.
Linux is based on the algorithm of the clock that associate a bit to be used and other to be modified with each of the main memory pages. The used in Linux is replaced by an 8-bit variable. Each time you access the page, the variable is increased. Linux runs periodically after the list of pages and decreases age of the variable in each page as you rotate through them in main memory. When a page has a 0 is an "old page", it has not been referred to in a while, and therefore it is the best candidate for replacement. When the value of the age is high, it is difficult to be replaced. The memory management kernel is made based on the page frames of main memory. Its basic function is to allocate and free frames for different uses. The basics of memory backup in Linux, are the reservation mechanisms of pages already used for user virtual memory management. When the buddy algorithm is used, you can reserve and release units of one or more pages. As the minimum size of memory that can be reserved is one page, the pages reserve would not be well suited as the core requires small pieces that will be used for a short time and are of different sizes. That's why Linux uses a scheme known as "assignment sheet". In a Pentium / x64 machine, the page size is 4 Kbytes and fragments within a page can be assigned in sizes 32, 64, 128, 252, 2040 and 4080 bytes.
